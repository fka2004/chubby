Mac加载内核提示签名失败

使用

加载自己编写的内核时报错

使用kextutil查看错误信息

解决方法：
1.重启mac后按下command+r进入恢复模式，终端输入命令
sudo nvram boot-args=kext-dev-mode=1
这命令的作用是关闭rootless功能，此功能的作用是通过签名验证应用。
这条命令貌似已经无效
2.依旧是恢复模式下，使用csrutil disable 关闭rootless功能
使用方法2成功解决

I/O Kit 笔记

#I/O Kit 详解

##什么是I/O Kit
I/O kit是系统framework，librarier,tools,和其他资源的集合，用来创建系统的驱动。

##开发所需应用和工具
xcode
I/O register Explorer
I/O注册表的内容和结构图形探索
Package Maker
为安装程序创建安装包，用于部署内核扩展

##驱动开发的命令行

###ioreg
显示I/O kit 注册表,以分层反转树形式展现

###kextload
加载内核扩展（例如设备驱动程序）或生成静态链接的符号文件以进行远程调试。
